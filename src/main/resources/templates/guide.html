چگونه از Master OAuth استفاده کنیم؟

این صفحه قدم‌به‌قدم توضیح می‌دهد چطور وارد شوید، scope انتخاب کنید، توکن بگیرید، API صدا بزنید و توکن را ابطال کنید. آدرس پایه فعلی: http://localhost:8089

۱. معرفی پلتفرم

Master OAuth یک Authorization Server است. کاربرد اصلی‌اش این است که پس از احراز هویت کاربر، یک JWT Access Token با scopeهای تأییدشده صادر کند تا سایر سرویس‌ها بتوانند با هدر Authorization: Bearer ... به APIها دسترسی داشته باشند.

  • لاگین مرورگر با کپچا
  • صفحه انتخاب scope (Consent)
  • صدور / لیست / ابطال توکن
  • CRUD کاربران و Scopeها
  • مستندات تعاملی Redoc در /docs

۲. ورود از صفحه لاگین

  1. به صفحه ورود بروید.
  2. نام کاربری و رمز را وارد کنید.
  3. کد تصویر کپچا را وارد کنید؛ در صورت نیاز روی تصویر کلیک کنید تا عوض شود.
  4. پس از ورود موفق به صفحه /home می‌روید.
اگر مستقیماً فقط لاگین کنید و به /oauth/authorize بدون client_id بروید، خطای invalid_client می‌گیرید. برای صدور توکن حتماً جریان OAuth را با لینک کامل authorize شروع کنید.

۳. جریان Authorization Code (پیشنهادی برای وب)

  1. این لینک را در مرورگر باز کنید (اگر لاگین نیستید، اول لاگین + کپچا می‌آید). در آدرس زیر HOST را با پایه بالا عوض کنید: 
    HOST/oauth/authorize?response_type=code&client_id=web-client&redirect_uri=http://localhost:8081/callback&scope=read%20write%20profile%20email&state=demo
    نمونه آماده: 
    http://localhost:8089/oauth/authorize?response_type=code&client_id=web-client&redirect_uri=http://localhost:8081/callback&scope=read%20write%20profile%20email&state=demo
  2. در صفحه Consent فقط scopeهایی که برای کاربر مجاز است نمایش داده می‌شود؛ موارد دلخواه را انتخاب و تأیید کنید.
  3. مرورگر به آدرسی شبیه این هدایت می‌شود: 
    http://localhost:8081/callback?code=XXXXXX&state=demo
    مقدار code را کپی کنید (یک‌بارمصرف و کوتاه‌عمر است).
  4. بلافاصله کد را به توکن تبدیل کنید: 
    curl -u web-client:password \
  -d "grant_type=authorization_code" \
  -d "code=CODE" \
  -d "redirect_uri=http://localhost:8081/callback" \
  http://localhost:8089/oauth/token
  5. در پاسخ JSON این فیلدها مهم‌اند:
    • access_token — برای هدر Authorization
    • jti — شناسه توکن برای ابطال
    • scope — scopeهای واقعی داخل توکن
    • refresh_token — تمدید توکن

از صفحه خانه هم می‌توانید دکمه «ادامه با انتخاب Scope» را بزنید تا همین جریان شروع شود.

۴. گرفتن توکن سریع (Password Grant)

برای تست API بدون مرورگر:

curl -u api-client:password \
  -d "grant_type=password&username=alice&password=password&scope=read write profile" \
  http://localhost:8089/oauth/token
توجه: حتی اگر در درخواست write بفرستید، فقط scopeهایی که هم در کلاینت و هم در کاربر مجاز باشند وارد توکن می‌شوند. مثلاً کاربری که فقط read دارد نمی‌تواند write بگیرد.

۵. استفاده از Access Token

توکن را در هدر بفرستید (نه jti):

curl -H "Authorization: Bearer ACCESS_TOKEN" \
  http://localhost:8089/api/me

نمونه‌های دیگر:

# پروفایل و اسکوپ‌های من
curl -H "Authorization: Bearer ACCESS_TOKEN" http://localhost:8089/api/me/scopes

# فهرست توکن‌های من
curl -H "Authorization: Bearer ACCESS_TOKEN" http://localhost:8089/api/tokens/me

۶. ابطال توکن (Revoke)

برای ابطال، در URL مقدار jti را بگذارید و در هدر همان access_token معتبر را بفرستید: 

curl -X POST \
  -H "Authorization: Bearer ACCESS_TOKEN" \
  http://localhost:8089/api/tokens/revoke/JTI_VALUE

ابطال همه توکن‌های کاربر جاری:

curl -X POST -H "Authorization: Bearer ACCESS_TOKEN" \
  http://localhost:8089/api/tokens/revoke-all

بعد از ابطال، همان توکن با پیام Token has been revoked رد می‌شود.

۷. مدیریت کاربران و Scope (CRUD)

نیاز به توکن دارای scope write:

# 1) گرفتن توکن
curl -u api-client:password \
  -d "grant_type=password&username=alice&password=password&scope=read write" \
  http://localhost:8089/oauth/token

# 2) ایجاد کاربر (ACCESS_TOKEN را از پاسخ بالا بگذارید)
curl -X POST -H "Authorization: Bearer ACCESS_TOKEN" -H "Content-Type: application/json" \
  -d '{"username":"charlie","password":"password","scopes":["read","profile"]}' \
  http://localhost:8089/api/users

# 3) ایجاد scope
curl -X POST -H "Authorization: Bearer ACCESS_TOKEN" -H "Content-Type: application/json" \
  -d '{"name":"reports","displayName":"گزارش‌ها","description":"دسترسی گزارش"}' \
  http://localhost:8089/api/scopes

# 4) فهرست کاربران
curl -H "Authorization: Bearer ACCESS_TOKEN" http://localhost:8089/api/users

فهرست عمومی scopeها بدون توکن: /api/scopes

۹. خطاهای رایج

خطاعلت معمولراه‌حل
invalid_client authorize بدون client_id یا secret اشتباه لینک کامل authorize یا web-client:password
invalid_grant کد منقضی یا دوبار مصرف شده دوباره authorize کنید و سریع code را عوض کنید
Cannot convert access token to JSON در هدر Bearer مقدار jti یا توکن ناقص گذاشته‌اید کل access_token را در هدر بگذارید
Token has been revoked توکن ابطال شده توکن جدید بگیرید
کپچا نامعتبر کد اشتباه یا منقضی تصویر را تازه کنید و دوباره وارد کنید
بازگشت به ورود مشاهده Redoc
